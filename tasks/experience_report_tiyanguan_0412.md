## 核心发现

### P0 问题（阻塞游戏）

**1. 游戏崩溃：NPC 对话触发 "'str' object has no attribute 'value'"**
- 影响：玩家第一次尝试与 NPC 对话时游戏直接崩溃，完全阻断社交/任务系统
- 复现：运行任何包含 NPC 对话的流程（如"和酒馆里的人说话"），错误日志：
  ```
  Error in subscriber 'game_master' for event player_input: 'str' object has no attribute 'value'
  NPC not found: None
  ```
- 根因分析：
  - `event_bus.py` 的 `_dispatch_event` 中调用 `sub.callback(event)`
  - `_on_player_input` 接收 Event 后，在处理链中某处将字符串当作枚举类型调用了 `.value`
  - 同时 `npc_agent.py` 的 `_on_npc_dialogue` 收到 `npc_id=None`，说明事件发布时 NPC ID 未正确传递
- 阻塞程度：**致命** — 游戏无法完成任何涉及 NPC 交互的任务

**2. 系统命令部分失效（同一输入有时有效有时无效）**
- 上一轮体验报告（`experience_report_20260412.md` 11:12场）显示"查看状态"和"查看背包"得了 8/10，但"任务/商店"在本次运行中直接崩溃
- 影响：玩家获得不一致反馈，不知道哪些命令可用

### P1 问题（影响体验）

**1. 模板滥用严重**
- 上一轮自动化测试（11:12场）：20 次模板使用
- 最高频模板："空气中弥漫着 XX 气息" — 几乎所有探索回复都出现
- 场景叙事缺乏具体性：酒馆场景出现自相矛盾的描述（"这里是一个酒馆"→"这里似乎没有人"→"没有值得探索的"）

**2. 空响应率极高**
- 上一轮测试（11:12场）：19 个动作中 9 个空响应（47%）
- 空响应分布在：NPC 对话（3次）、场景切换（1次）、战斗触发（3次）、边界输入（3次）

**3. 战斗系统不稳定**
- 某些测试中"向酒馆老板打听消息"意外触发战斗叙事（Q:8 但不应该在打听消息时触发战斗）
- 战斗触发条件模糊，玩家无法预测何时进入战斗
