# 记忆管理系统

## 分层记忆

### 短期记忆（Short-term）

- 当前 session 内的上下文
- 当前 Beat 的详细内容
- 玩家在本 session 的即时动作

### 长期记忆（Long-term）

- 历史事件、人物关系
- 剧情进度、已解锁区域
- 玩家画像（习惯、性格）

---

## 检索方式

**RAG（检索增强生成）**：
- 玩家动作时自动查询相关记忆
- 用工具调用而非全量塞 context
- 减少 token 消耗，提高响应质量

---

## 记忆写入时机

- 每个 Beat 结束时，自动生成本 Beat 的 `Beat Summary`
- 重大事件（剧情转折、角色死亡、重要发现）立即写入
- 玩家明确的重要决策，记录到长期记忆

---

## 技术实现

- Markdown 格式存储（初期）
- 后期扩展：SQLite 存储结构化数据 + 向量数据库做语义检索
- 场景内容按 `/world/` 目录结构组织，支持按类型和标签检索
