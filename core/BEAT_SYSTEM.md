# 大回合（Beat）机制

## 核心概念

**Beat（叙事节拍）**：游戏中的基本时间单位，一个 Beat 内包含完整的"叙事启动 → 玩家行动 → 叙事收尾"循环。

无论是探索场景还是战斗场景，统一用 Beat 作为回合单位。

---

## Beat 生命周期

```
【Beat 开始】
    主DM：根据上回合结果 + 规划
          → 向所有玩家统一叙述当前背景/事件
          → 向各子Agent 发送本回合的上下文和交互边界

【Beat 进行中】
    各子Agent：在各自窗口内与玩家对话
    主DM：旁观，不干预

【Beat 结束触发】
    触发条件（满足任一）：
    - 玩家主动表示结束（"我搞定了"）
    - 玩家点击"本回合结束"按钮
    - 达到子Agent设定的交互边界

【Beat 结算】
    子Agent：汇报本轮交互结果给主DM
    主DM：根据所有汇报 → 做出全局事件裁决
    → 进入下一个 Beat
```

---

## 子 Agent 交互边界

### 边界定义

主DM 在 Beat 开始时设定边界，例如：
- "这回合你最多可以调查 3 个地点"
- "这回合你可以与 2 个 NPC 对话"

### 实时提醒

子Agent 在对话过程中需实时提醒玩家：
- "你还可以调查 2 个地点"
- "这回合快结束了，你还有什么想做的吗？"

避免玩家觉得突然结束。

### 边界触发规则

| 触发条件 | 说明 |
|----------|------|
| 玩家主动结束 | 玩家明确表示"搞定了" |
| 按钮点击 | 玩家点击 UI 的"结束本回合"按钮 |
| 达到上限 | 消耗完主DM设定的交互次数 |
| 子Agent判定 | 子Agent认为当前对话已达合理收尾点，可主动引导结束 |

---

## 探索模式 vs 战斗模式

### 探索模式（宽松回合制）

- 玩家自由发言、行动
- 子Agent 做为主要对话接口
- 主DM 根据互动内容裁决结果

### 战斗模式（严格回合制）

- 严格先攻顺序（Initiative Order）
- 轮流行动，一轮（Round）内所有玩家/NPC 依次操作
- 主DM 控制战场形势和规则裁决
- 战斗结果触发状态变更，同步到全局状态

> 注：两种模式统一在 Beat 框架下，差异仅在规则严格程度，由主DM 根据场景切换。

---

## 技术实现要点

- 每个 Beat 结束时，主DM 需要输出结构化的 `Beat Summary`
  - 本 Beat 发生的关键事件
  - 各玩家状态的变更
  - 下一个 Beat 的起点状态
- 子Agent 的汇报也需要结构化，便于主DM 聚合处理
- Beat Summary 作为记忆单元，自动写入 session memory
