# 核心架构

## 双层 Agent 架构

```
主 DM Agent（全局控制）
├── 主导剧情走向
├── 全局状态管理
├── 伏笔触发控制
├── 规则验证（通过 Hook）
└── 向所有玩家广播剧情结果

子 Agent（每个玩家一个）
├── 与玩家直接对话
├── 解析玩家输入（动作类型识别）
├── 总结玩家画像（习惯、性格）
├── 向主 Agent 上报有意义的动作
└── 接收主 Agent 决策并反馈给玩家
```

### 协作流程

```
玩家输入 → 子Agent接收 → 动作解析 + Hook验证 → 上报主Agent
主Agent决策 → 事件触发/状态更新 → 结果传回子Agent
子Agent → 生成自然语言 → 反馈给玩家
```

## 事件总线 + Hook 机制

所有跨模块通信通过事件总线实现，支持以下 Hook：

| 事件 | 触发时机 | 监听器 |
|------|----------|--------|
| `player_action` | 玩家做出动作 | 规则验证 Hook |
| `memory_query_needed` | 需要查记忆时 | 记忆检索 Hook |
| `plot_trigger_check` | 每次动作后 | 伏笔触发 Hook |
| `state_change` | 状态变更时 | 状态持久化 Hook |
| `dice_roll` | 需要骰子时 | 随机数 Hook |
| `npc_dialogue` | NPC 对话时 | 人设校验 Hook |

### Hook 的重要意义

- LLM 不需要判断"何时验证/触发"，由系统根据动作类型自动决定
- 新增 Hook 只需注册，不影响核心逻辑
- 方便调试和测试

### 接口 vs 自动触发的边界

- 模糊地带，实际编码时再决定
- 原则：需要 LLM 判断的做成接口，系统能判断的自动触发

## 目录结构

```
/ai-dm-rpg/
├── /world/           # 世界观、场景、NPC 描述
├── /events/          # 剧情事件、伏笔触发器
├── /characters/      # 玩家角色、NPC 模板
├── /state/           # 运行状态（剧情进度、各角色状态）
├── /memory/          # 记忆检索用的内容块
├── /tools/           # 工具函数定义（规则库、状态读写接口）
├── /sessions/        # 每次跑的 session 日志
└── /assets/          # 静态资源（图片、音频等）
```
