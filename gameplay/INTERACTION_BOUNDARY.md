# 子 Agent 交互边界

> 定义子 Agent 与玩家在一个 Beat 内的交互规则，确保玩家有清晰的游戏节奏感，不会莫名其妙突然结束。

---

## 边界定义方式

**主 DM 在 Beat 开始时设定**，例如：
- "这回合你最多可以调查 3 个地点"
- "这回合你可以与 2 个 NPC 对话"
- "这回合你可以进行 1 次战斗"

---

## 结束触发条件

满足任一条件，本 Beat 结束：

| 触发条件 | 说明 |
|----------|------|
| 玩家主动结束 | 玩家明确表示"我搞定了" |
| 按钮点击 | 玩家点击 UI 的"结束本回合"按钮 |
| 达到上限 | 消耗完主 DM 设定的交互次数 |
| 子 Agent 主动引导 | 子Agent 认为当前对话已达合理收尾点 |

---

## 实时提醒机制

子 Agent 在对话过程中必须提醒玩家：

**提醒时机**：
- 交互次数剩余 1 次时
- Beat 即将结束时（倒计时）

**提醒方式**：
- 自然融入对话，例如：
  > "你还可以调查 2 个地点。村口的老槐树下似乎有人在窃窃私语..."
  > "这回合快结束了，你还有什么想做的吗？"

---

## 防止突兀结束

- 每次玩家完成一个动作后，子 Agent 应主动提示当前状态
- 避免突然出现"本回合结束"——给玩家准备时间
- 鼓励玩家在结束前主动确认是否有遗漏

---

## 技术要点

- 子 Agent 状态需包含当前 Beat 的 `remaining_actions` 计数
- 每执行一次交互动作，计数 -1
- 计数归零时，自动触发结束流程
