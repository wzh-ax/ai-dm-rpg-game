# 节奏控制层

> 负责监测和调节游戏节奏，确保玩家体验"跌宕起伏"，而非平淡无奇。

---

## 问题背景

LLM 生成的内容天然倾向于"平稳"——连贯合理，但不太主动制造冲突和高潮。

没有节奏控制时，生成的地图可能是：
> 出发 → 走一会儿 → 看到有趣的建筑 → 进去看看 → 出来 → 继续走 → 看到另一个有趣的地方 → 结束

整条线都是"平行"的，没有波动。

---

## 节奏控制架构

```
┌──────────────────────────────────────┐
│         Pacing Controller            │
│  ┌──────────────────────────────┐    │
│  │  节奏监测模块                │    │
│  │  - 分析最近 N 个事件的类型   │    │
│  │  - 检测节奏是否过于单一      │    │
│  └──────────────────────────────┘    │
│  ┌──────────────────────────────┐    │
│  │  节奏注入模块                │    │
│  │  - 主动触发节奏变化事件      │    │
│  │  - 下发特定类型事件要求      │    │
│  └──────────────────────────────┘    │
└──────────────────────────────────────┘
           ↓↑
┌──────────────────────────────────────┐
│         内容生成层（SCENE_GEN）       │
│  - 接收 Pacing 的指令               │
│  - 生成符合节奏要求的内容            │
└──────────────────────────────────────┘
```

---

## 节奏参数

```json
{
  "pacing": {
    "wave_interval": 5,
    "high_intensity_ratio": 0.3,
    "low_intensity_ratio": 0.2,
    "surprise_probability": 0.15
  }
}
```

| 参数 | 说明 | 示例值 |
|------|------|--------|
| `wave_interval` | 每 N 个事件注入一次节奏变化 | 5 |
| `high_intensity_ratio` | 高强度事件占比（战斗/危机） | 0.3 |
| `low_intensity_ratio` | 低强度事件占比（休息/社交） | 0.2 |
| `surprise_probability` | 突然转折概率 | 0.15 |

---

## 地图基调

每张地图定义一个情感基调，引导 LLM 生成方向：

| 基调 | 说明 |
|------|------|
| `tense` | 紧张悬疑，充满未知危险 |
| `epic` | 史诗冒险，宏大叙事 |
| `mysterious` | 神秘探索，迷雾重重 |
| `survival` | 生存压力，资源匮乏 |

---

## 节奏锚点

在地图生成时预设结构化锚点，不决定具体内容，只决定"这里要有起伏"：

```
旅程段：
  - Anchor_High: 遭遇战 / 意外发现 / 强力NPC
  - Anchor_Low: 休息 / 友好互动 / 获取线索
  - Anchor_Twist: 看似好事变坏事，或反之
```

---

## 玩家反馈闭环

```
玩家反馈收集（副本结束后）
    ↓
反馈数据存储（参数 + 评分）
    ↓
A/B 测试 + 迭代优化
    ↓
找到"最优参数"组合
```

**收集维度**：
- 哪些时刻感到无聊/紧张/兴奋
- 整体节奏评分（1-5）
- 地图基调标签

> 注：MVP 阶段使用固定参数，后期叠加反馈优化系统。
