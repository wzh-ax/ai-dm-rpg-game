# 场景四步生成流程

> 当玩家探索到未定义区域时，触发此流程。目标是：**先定性，再生成**，确保每个场景有差异化，而非千篇一律。

---

## 触发时机

**懒生成原则**：
- 玩家推进到某节点 → 查询现成内容
- 有现成内容 → 直接使用
- 没有现成内容 → 触发四步生成
- **不递归向下扩展**——只生成玩家即将到达的那个节点

---

## 四步生成流程

### Step 1：场景登记查询

- 查询同类场景索引（如都是 `forest` 类型）
- 提取已有场景的核心标签（避免重复）
- 输出：已有标签组合列表

### Step 2：差异化定位（LLM）

- 输入：已有场景标签列表
- 任务：定义新场景的核心思想，要求与已有场景区分
- 输出：差异化定位描述

### Step 3：场景纲要生成（LLM）

- 基于差异化定位，生成 3-5 个特色元素
- 定义氛围、危险等级、潜在事件类型
- 输出：场景纲要（约 50-100 字）

### Step 4：详细内容生成（LLM）

- 基于纲要细化，生成具体描述
- 包括：环境描述、常驻 NPC（可选）、可触发事件

---

## 场景元数据

每个生成场景注册到索引时，必须包含：

```json
{
  "id": "forest_north_village_001",
  "type": "forest",
  "core_concept": "被遗忘的古战场遗址，树木从枯骨中生长",
  "tags": ["haunted", "ancient_battle", "gloomy", "mid_danger"],
  "unique_features": ["会低语的枯叶", "地下沉睡的兵器"],
  "纲要": "..."
}
```

---

## 差异化机制

### 标签差异化

确保新场景至少有一个标签与其他同类场景不同。

### 核心概念查重

LLM 自我判断新生成的核心思想是否与已有雷同，若相似度高于阈值，要求重新生成定位。

### 不重复原则

同类场景不生成相同核心：
- 森林 A：古战场遗址
- 森林 B：精灵隐居地
- 森林 C：沼泽化森林

---

## 示例

**场景需求**：玩家探索"村庄北边的森林"

1. **查询索引**：发现已有 `forest` 类型场景：森林 A（古战场）
2. **差异化定位**：新森林核心 = 迷雾中的精灵遗迹
3. **纲要生成**：
   ```
   雾气笼罩的古老森林，
   精灵族撤离后留下的遗迹，
   有微弱的魔法残留，偶尔触发幻象
   ```
4. **详细内容**：生成迷雾、精灵废墟、魔法残留等具体描述
