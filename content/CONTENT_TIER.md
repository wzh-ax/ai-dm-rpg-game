# 内容分级与审核流程

## 四层内容体系

```
┌─────────────────────────────────────────────────────┐
│  宪法层（System Prompt）                             │
│  World Constitution 定义铁律，全局约束                 │
├─────────────────────────────────────────────────────┤
│  结构化边界层（Hook 校验）                            │
│  regions.json 定义存在/不存在，LLM 生成时自动拦截      │
├─────────────────────────────────────────────────────┤
│  核心内容层（Canon - 人工审核）                        │
│  重要城市、关键剧情节点、人工审核后写入                │
├─────────────────────────────────────────────────────┤
│  动态内容层（自动生成）                               │
│  填充型场景、随机事件，LLM 自由生成                   │
└─────────────────────────────────────────────────────┘
```

---

## 内容分级

| 级别 | 说明 | 审核方式 | 示例 |
|------|------|----------|------|
| **Canon** | 核心内容，影响主线 | 人工审核后入库 | 主城布局、关键 NPC |
| **Verified** | 重要但不关键 | AI 初筛 + 人工复核 | 支线剧情、非关键地点 |
| **Generated** | 填充型内容 | AI 自动生成 | 路边风景、随机遭遇 |
| **Temporary** | 临时内容 | 不入库，仅当次使用 | 玩家探索时生成但未触发审核 |

---

## 审核流程

### Canon 入库流程

```
LLM 生成 → 标记为 Candidate
         → 人工审核（价值/逻辑/一致性）
         → 通过：写入 /world/canon/
         → 不通过：返回修改或丢弃
```

### 异步审核

- 生成内容先存储为 `candidate` 状态
- 定时或批量由人工/AI 审核
- AI 初筛：判断是否符合 Constitution + 逻辑一致性
- 人工复核：模糊案例最终确认

---

## 内容过滤三层

```
输入层（硬限制）
  └─ 正则/关键词匹配，拦截违法内容

输入层（软限制）
  └─ 血腥暴力词汇替换，让 LLM 输入干净

输出层（System Prompt 约束）
  └─ 分级标注事件级别，约束描写边界
```
